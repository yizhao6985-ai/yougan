# 各平台创作要点

有感从**定方案阶段**就嵌入平台信息，让结构、语气、篇幅更贴近发布渠道。以下是在有感中创作时的平台向建议（非各平台官方规则，发布前请以平台最新规范为准）。

## 通用：定方案时写明平台

在对话中直接说明目标平台，例如：

> 主要发小红书，笔记体，800 字以内，带 3 个产品对比。

方案面板的「制作规格」会记录平台与体裁，制作环节会据此选用相应指引。

## 小红书

### 适合的内容形态

- 图文笔记、清单种草、教程步骤、避坑合集
- 短段落、多 emoji（适度）、强首图思维

### 定方案建议

- **受众**：年龄、城市层级、消费场景要具体
- **结构**：钩子 → 干货主体 → 总结/互动
- **规则**：注明是否报备、是否对比竞品、字数上限

### 对话示例

> 小红书护肤笔记，面向干皮通勤族，清单体 5 条建议，语气像朋友分享，不要医疗承诺。

### 制作后检查

- 标题是否能在信息流里一眼看懂收益
- 正文是否可扫读（小标题、换行）
- 标签是否贴近搜索词

## 微信公众号

### 适合的内容形态

- 深度长文、专栏、故事、行业观察
- 逻辑链完整，适合分段阅读

### 定方案建议

- **结构**：引子 → 问题 → 分析/方法 → 结语
- **语气**：可更正式；仍建议短段适配手机
- **参数**：目标字数（如 2000–3000 字）

### 对话示例

> 公众号长文，主题远程办公效率，受众中层管理者，专业但不晦涩，四个方法加案例。

## 微博

### 适合的内容形态

- 短帖、热点评论、金句+链接长文导流
- 信息密度高，开头即观点

### 定方案建议

- 明确是 **短帖动态** 还是 **长文链接体**
- 规则里可写「不超过 140 字」或线程条数

## 抖音 / 快手 / 哔哩哔哩

有感当前以 **脚本、口播稿、分镜文案、选题策划** 为主，替代专业剪辑软件。

### 定方案建议

- 体裁选「视频脚本」或「口播稿」
- 结构：前 3 秒钩子 → 价值点 → 行动号召
- 参数：时长（如 60 秒）、横竖屏、人设语气

### 对话示例

> 抖音 60 秒口播，教新手做咖啡，口语化，分镜按秒标注。

## 发布到有感「发现灵感」

若暂无第三方账号或想先积累作品集：

- 目标平台可设为「有感」
- 发布后出现在 `/content`，可按体裁、主题被他人发现
- 适合练笔、存稿、建立个人主页作品集

## 多平台同一选题

**建议每件作品对应一个主平台**。若同一选题要发多平台：

1. 完成主平台版本
2. **另存为新作品**
3. 在新作品里说「改成公众号长文语气/微博短帖体」再制作

避免一个方案里混多个平台规则导致成稿四不像。

## Pro 与第三方发布

Pro / Pro+ 可绑定小红书、微博、公众号、抖音、快手、B 站等账号（需运营方配置 OAuth）。绑定后可在成稿后减少复制粘贴步骤。详见 [11-account-and-settings.md](./11-account-and-settings.md)。
