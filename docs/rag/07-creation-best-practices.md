# 创作好作品的方法与技巧

## 核心原则：先定方向，再出正文

有感的设计基于一个观察：**选题和结构偏了，后面写得再好也要重写**。

因此最有效率的用法是：

1. 用 2–5 轮对话把方案定稳（主题、平台、受众、结构）
2. 再说「开始制作」
3. 成稿后只做局部修改，而非推倒重来

这与「一句 Prompt 出一篇」的工具相反，但更适合**周更、日更**的创作者。

## 减少返工的 5 个技巧

### 1. 在方案里写清「不要什么」

制作规则里的否定约束往往比泛泛的「写好一点」更有效：

- 「不要夸大疗效」
- 「不要用震惊体标题」
- 「不要超过 3 个产品对比」

### 2. 一次只改一类问题

改稿时尽量聚焦，例如：

- 第一轮只改标题
- 第二轮只改语气
- 第三轮只改长度

一次提太多修改点，容易互相打架。

### 3. 善用「提问」学方法，用「制作」出成品

| 需求 | 做法 |
|------|------|
| 学写法、要建议 | 直接问：「小红书清单笔记有什么结构？」 |
| 要写入方案 | 说：「把刚才说的结构记入方案」 |
| 要出正文 | 说：「开始制作」或具体改稿句 |

### 4. 对照作品面板的四个标签

养成习惯：聊完看一眼 **方案 → 参考 → 作品 → 版本**，确认 AI 理解与你的意图一致。

### 5. 大方向变了就新开作品

换平台、换选题、换体裁时，**另存为新作品**比在同一件作品里硬改更干净。

## 各体裁简要要点

### 清单笔记（小红书常见）

- 开头 1–2 句戳痛点或给结果预览
- 主体 3–7 条，每条可扫读（小标题 + 一两句）
- 结尾总结或互动问句
- 标签 3–5 个，贴近搜索词而非堆砌

### 观点长文（公众号常见）

- 开头用故事、反常识或数据抓住读者
- 中间 2–4 个论证层次，每层有小标题
- 结尾落到可行动的建议或立场
- 段落宜短，适合手机阅读

### 案例故事

- 人物/场景要具体（时间、地点、细节）
- 冲突 → 转折 → 收获 的结构清晰
- 避免流水账，每段推进一个信息点

### 教程干货

- 先写「学完能做什么」
- 步骤编号，一步一图或一例证更佳
- 预告难度和前置条件（适合谁、需要什么基础）

### 对比评测

- 先说评测维度和立场（是否收费、是否真实购买）
- 表格或并列结构帮助扫读
- 结论要明确，避免「都好都好」

## 创意度怎么选

| 内容类型 | 创意度建议 |
|----------|------------|
| 教程、事实、新闻解读 | 偏低 |
| 种草、故事、需要新鲜角度 | 中等 |
| 头脑风暴、多版标题 | 偏高 |

创意度只影响制作环节，不影响方案整理。

## 用「发现灵感」学习他人

浏览 `/content` 公开作品时，可以：

- 看同平台、同体裁的标题和结构
- 用筛选找「干货」「故事」「笔记」等类型
- 回到自己作品里用参考或对话复现学到的结构（注意原创）

## 提问环节可以问什么

- **优化**：「这篇标题弱点在哪？」「如何让开头更有钩子？」
- **学习**：「什么是内容节拍？」「公众号和小红书语气差在哪？」
- **背景**：「这个受众最关心什么？」「这个品类有什么合规红线？」

需要把答案固化进方案时，补一句「记入方案」即可。

## 有感帮不了你什么时要自知

- AI 不能替代你的真实体验和专业判断（评测、医疗、法律等需你负责）
- 不能保证一定爆款；方案与执行提高的是**可控性和效率**
- 平台算法和热点变化快，内容仍需你根据数据迭代
