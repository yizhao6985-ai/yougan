# 03 · 产品与解决方案素材

供 BP「解决方案与产品 / 产品与服务」章节使用。事实见 [00-fact-sheet.md](./00-fact-sheet.md)。

## 1. 产品定位【已核验】

- **品类**：AI 自媒体创作助手
- **落地页主张**：AI 创作助手
- **应用内主张**：方案 · 制作 · 提问
- **一句话**：先定方案，再由 AI 团队按计划做出能发布的作品

## 2. 分阶段创作方法论（核心）【已核验】

```text
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  定方案  │──▶│  备参考  │──▶│  制作    │──▶│  作品    │──▶│   发布   │
│ profile  │   │reference │   │production│   │ preview  │   │ 有感/平台│
└──────────┘   └──────────┘   └────┬─────┘   └──────────┘   └──────────┘
      │              │              │
      └─ 侧栏直改 PATCH ─┴─ API 同步 thread ─┘
```

每条用户消息经**回合队列**自动路由到对应环节子图；同一作品内可随时切换。详见 `docs/technical/agent-turn-queue.md`。

**标准作业程序（SOP）**

| 步 | 用户动作 | 系统产出 |
|----|----------|----------|
| 01 | 新建作品（开屏给 7 条下一步建议） | 作品 + LangGraph thread |
| 02 | 定方案对话或侧栏直改 | 侧栏「方案」（direction / style / setting / requirements / bounds） |
| 03 | 上传参考或维护借鉴意图 | 侧栏「参考」（多模态分析入库） |
| 04 | 说「开始制作」，AI 团队编排计划 | 内部 production plan |
| 05 | 按任务执行（文字/视觉/音视频） | 作品预览 preview（含文生图配图） |
| 06 | 发布 | 有感公域 / 第三方平台（Pro/Pro+） |

## 3. 创作台（Studio）【已核验】

布局：左侧作品列表与分组 → 中间对话区 → 右侧「作品面板」。

| 面板 Tab | 功能 |
|----------|------|
| 对话区 | 回合队列自动路由、流式回复、工具调用可视化、回合末/开屏可点击建议 |
| 方案 | profile 步骤向导：固定设定、结构大纲、规则；可对话或侧栏直改 |
| 参考 | references：文本/图片/音频/视频参考，经分析汇总 |
| 作品 | preview：标题、正文、配图图册 |
| 版本 | 单线版本时间轴，可回到任意版本、从某版另存为新作品 |

辅助能力：创意度滑杆（0.1–1.0，仅影响制作环节）、作品分组、云端同步、开屏 7 条下一步建议。

## 4. 文生图（视觉制作）【已核验 · 新能力】

设计任务从「写 prompt」到「成图入库、前端展示」的完整链路（`docs/technical/design-image-generation.md`）：

- 出图画幅由作品方案（平台/体裁）推断，保证文案与配图一致；
- Agent 用 MiniMax `image-01` 出图（仅产出临时外链）；
- **API 在写库前把图片物化到自有存储（OSS/local）**，前端始终读稳定 URL；
- 前端缩略图 + 全屏图册展示。

商业含义：图文创作者「文案 + 配图」一站式完成，提升单作品价值与平台适配度。

## 5. 内容与社区：发现灵感【已核验】

- **发现灵感**（`/content`）：浏览创作者公开发布作品，按平台/体裁/主题/媒介四维筛选，用于选题参考与获客。
- **个人主页**：展示已发布作品、总阅读量、近 6 月发布趋势，支撑作者品牌。
- **增长飞轮**：创作 → 发布到有感 → 公域曝光 → 访客注册 → 成为创作者。

## 6. 平台集成发布【已核验】

支持绑定并发布至 9 个平台：小红书、微博、Twitter/X、LinkedIn、Instagram、微信公众号、抖音、快手、哔哩哔哩。

- 绑定走 OAuth（令牌存 `PlatformIntegration`）；
- 「发布到有感」公域无需 OAuth，已实现；
- 第三方一键发布 OAuth 框架已具备，按平台分批配置上线，属 Pro/Pro+ 权益；
- 价值主张：**少一道复制粘贴**，在 Studio 完成几步后直达粉丝所在平台。

## 7. 多端规划

- **Web**：当前主阵地，完整流程 + 设置 + 会员。
- **手机 App**（规划中）：通勤/睡前快速记灵感，回 Web 同一作品继续创作与出稿；账号与作品同步（介绍页已上线，下载链接待配置）。

## 8. 技术架构摘要【已核验】

```text
浏览器 (Web :3000) → API 中间层 (:4000，鉴权/作品/发布/订阅/上传/LangGraph 代理)
   ├─ Postgres（:5432 — yougan_api 业务库 + yougan_agent checkpoint）
   ├─ Redis（可选缓存 :6379）
   └─ Agent (:2024，LangGraph：回合队列 → reference/profile/production/ask 子图)
```

- **LLM**：阿里百炼 DashScope（OpenAI 兼容）+ MiniMax；主对话 `qwen3.7-max`、多模态 `MiniMax-M3`、文生图 `image-01`。
- **架构特点**：业务库与 AI checkpoint 同 Postgres 实例、分库；OpenAPI 驱动前后端契约，便于后续 API 开放与白标扩展；存储支持本地或阿里云 OSS。
