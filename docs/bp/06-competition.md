# 06 · 竞争分析素材

供 BP「竞争分析」章节使用。

---

## 1. 竞争格局

| 类型 | 代表 | 与有感关系 |
|------|------|------------|
| 通用大模型对话 | ChatGPT、豆包、Kimi、文心等 | 替代：灵活但无作品 SOP；有感降低 Prompt 成本与返工 |
| 一键 AI 写作 | 各类营销文案 / 写作 SaaS | 部分替代：更快但易返工；有感强调前期确认 |
| 多平台分发工具 | 排版、定时、数据统计类 | 互补：分发强、创作弱；有感做创作前端 |
| AI 绘画工具 | Midjourney、即梦等 | 部分互补：有感把文案与配图合到同一作品流程 |
| 笔记 / 协作 | Notion、飞书文档 | 弱替代：通用文档 ≠ 自媒体流水线 |

## 3. 核心差异化（一句话）【已核验】

> 有感不是「更快的 AI 写字机」，而是「把选题、结构和成稿分开确认的生产线」；确认之后再出稿（含配图），并连接到发布。

支撑事实：定方案不直接出稿、制作不跳过任务记录、提问不擅改作品与计划；文案与配图同一流程；9 平台发布出口。

## 4. SWOT

| | 内容 |
|---|------|
| **S 优势** | 三步方法论清晰且由代码保证；作品面板 + 版本时间轴可回溯；文案+文生图一体；按成本计量抗模型价格波动；公域发现具生态雏形；技术栈现代、双库隔离、OpenAPI 契约 |
| **W 劣势** | 真实支付未闭环；平台发布依赖逐个运营配置；品牌与流量早期；App 未上架；内容审核与合规待建 |
| **O 机会** | 创作者基数大、平台教育成熟；图文配图刚需；MCN/模板/B 端与 API 开放/白标扩展空间 |
| **T 威胁** | 平台自带 AI；大模型降价压缩付费意愿；平台 API/OAuth 政策收紧；同质化价格战 |

## 5. 护城河建设路径

1. **流程与数据**：积累高转化的方案结构与选题模式，驱动推荐与模板。
2. **生态**：发现页 UGC + 作者主页的网络效应。
3. **分发绑定**：OAuth 账号绑定提高迁移成本。
4. **垂类深耕**：先打透 1–2 个平台（建议小红书）再横向扩展。
5. **成本工程**：多模型抽象 + 计量体系，持续优化次均成本形成价格优势。
