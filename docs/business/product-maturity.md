## 已验证的产品能力（可演示）

适合融资 BP 演示录屏的路径：

1. 注册 → 登录 → 新建作品。
2. 灵感推荐 → 灵感模式确认 2–3 条需求。
3. 切换创作模式 → 创意总监生成/更新制作计划 → 定稿。
4. 创作模式出稿 → 侧栏预览 → 调节创意度再改一版。
5. 发布到有感 → 发现页可见 → 主页阅读量增加。
6. 会员页查看额度 → 模拟开通 Pro → 额度变为 500。

## 关键缺口（Gap）

### P0 — 商业化闭环

| 缺口 | 影响 | 建议里程碑 |
|------|------|------------|
| 真实支付（微信/支付宝） | 无法产生真实收入 | M1–M2 |
| 支付回调与对账 | 财务合规 | 与支付同步 |
| 发票（如需 ToB） | 企业客户 | M3+ |

### P1 — 增长与留存

| 缺口 | 影响 | 建议 |
|------|------|------|
| 手机 App 上架 | 多端故事不完整 | 与移动端团队排期 |
| 至少 1 个平台 OAuth 端到端发布 | Pro 卖点难验证 | 优先小红书 |
| 邮件服务生产配置 | 找回密码/改邮箱体验 | 上线前必配 |

### P2 — 生态与安全

| 缺口 | 影响 | 建议 |
|------|------|------|
| 公开内容审核 | 合规与品牌风险 | 公域流量放大前 |
| 用户协议与隐私政策页 | 上线合规 | 法务文档 |
| 关注/互动 | 社区粘性弱 | 生态二期 |

### P3 — 差异化深化

| 缺口 | 影响 | 建议 |
|------|------|------|
| Pro「增强出稿」技术差异化 | 付费理由偏额度 | 模型/上下文配置区分 |
| 按量加购包 | 收入天花板 | 额度用尽后的 ARPU |
| 数据分析看板（创作者） | 运营价值 | 发布后数据回流 |

## 技术债务与商业风险

| 风险 | 描述 | 缓解 |
|------|------|------|
| LLM 供应商依赖 | MiniMax + DeepSeek | 抽象 `llm/` 工厂，预留切换 |
| 按次计费 vs token 成本波动 | 毛利波动 | 监控单次对话 token；调价机制 |
| 模拟支付误导测试数据 | 演示与真实环境分离 | 生产关闭 instant-paid |
| Agent 与 API 双库 | 运维复杂度 | 托管 Postgres、备份策略 |
| LangGraph 版本升级 | 兼容性 | 锁定版本、回归测试 |

## 建议路线图（商业计划用）

### 阶段一：验证 PMF（0–3 个月）

- 上线真实支付，验证免费→Pro 转化。
- 完成小红书 OAuth 发布闭环（单平台打透）。
- 配置生产 SMTP、OSS、域名与 HTTPS。
- 招募 50–100 名种子创作者，追踪 M2 制作计划定稿率。

### 阶段二：增长（3–6 个月）

- 发现页运营：精选、话题活动。
- 扩展 2–3 个平台 OAuth。
- 手机 App MVP（记灵感 + 同步）。
- 按量加购包。

### 阶段三：规模化（6–12 个月）

- MCN/团队版席位。
- 行业模板市场。
- 创作者数据看板（阅读、发布效率）。
- 可选：API 开放、白标。
