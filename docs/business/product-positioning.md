# 产品定位与价值主张

## 产品名称

- **中文品牌**：有感
- **英文品牌**：Yougan
- **品类描述**：AI 自媒体创作助手
- **一句话定位**：帮你收集灵感、制定制作计划，再按计划生成能发的图文内容。

## 要解决的核心问题

自媒体创作者在日常更新中常遇到三类痛点：

1. **没选题**：不知道写什么、面向谁、用什么形式。
2. **方向不稳**：通用 AI「一问一答」直接出全文，选题偏差就要整篇重写，浪费额度与时间。
3. **创作状态难回溯**：在多个工具或对话里零散修改，方案与成稿难以对齐。

有感将创作拆成**可确认、可回溯**的多模式工作流，让 AI 在每一步只做该做的事，人始终在关键节点拍板。

## 价值主张

| 维度 | 有感提供的价值 |
|------|----------------|
| 过程可控 | 定方案 → 备参考 → 制作分阶段，每步有侧栏「作品面板」沉淀，可随时回看与修改 |
| 减少返工 | production 子图先编排制作计划再出稿；严格按已定任务执行 |
| 体裁适配 | 从定方案阶段确认体裁、受众、语气与结构 |
| 作品化管理 | 一件作品 = 多轮对话共享长记忆，可分组，云端同步，换设备可续写 |
| 发布闭环 | 内容可发布到「发现灵感」公域，积累作品集与曝光 |

## 产品 Slogan（来自代码文案）

- 落地页：**AI 自媒体创作助手**
- 应用内：**方案 · 制作 · 提问**（聊天改状态走对话子图；侧栏直改经 API 同步 thread）
- 能力页主张：**从找灵感到发出内容**

## 与「一次性生成全文」类工具的差异

代码与文案中明确强调的对比点：

- 许多 AI 工具「问一句出一篇」，选题不准就要整篇重写。
- 有感拆成阶段：**先定选题和需求 → 创意总监制定制作计划 → 按计划生成标题和正文**。
- 同一件作品内可随时切换模式，界面、快捷键或对话指令均可，状态与侧栏即时对齐。

## 产品边界（当前版本不做的事）

根据 Agent 与各模式 prompt 归纳：

| 阶段 | 明确不做 |
|------|----------|
| 定方案（profile） | 不直接产出作品内容；不执行制作任务 |
| 备参考（reference） | 只维护参考素材；不改方案与成稿 |
| 制作（production） | 不跳过任务记录直接生成 |
| 提问（ask） | 不直接修改 preview / productionPlan；答疑与咨询为主 |
| 侧栏 PATCH | 不经 Agent run；更新 DB 并同步各对话 thread checkpoint |

这一定边界既是用户体验设计，也是差异化叙事的核心。

## 商业计划书可引用的摘要段落

> 有感（Yougan）面向需要持续产出图文与多媒体内容的创作者，用「定方案 → 备参考 → 按计划制作」的 AI 工作流替代一次性全文生成，降低选题偏差带来的返工成本，并通过作品云同步与公域内容发现，覆盖从构思到发布的关键环节。
