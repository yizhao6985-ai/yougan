## 1. 执行摘要

### 1.1 我们在做什么

**有感（Yougan）** 是一款面向自媒体创作者的 **AI 创作助手**。产品将内容生产拆解为可确认、可回溯的三步流水线——**灵感 → 大纲 → 创作**——帮助用户在定稿结构之后再生成标题与正文，从而降低「选题偏差导致整篇重写」的返工成本，并支持将成稿发布到有感公域或（Pro 会员）绑定的小红书、公众号等第三方平台。

### 1.2 为什么现在

- 中国图文/短内容创作者规模持续增长，平台算法驱动**高频更新**成为常态。
- 通用大模型「一问一文」普及，但**过程不可控、返工成本高**，与自媒体「先定选题与结构再写稿」的真实工作流错位。
- 多平台运营使「写完 → 复制 → 各平台后台发布」的链路摩擦突出，工具链分散。

### 1.3 商业模式摘要

| 项目 | 内容 |
|------|------|
| 收入模式 | SaaS 订阅（主）+ 未来加购包 / 团队版 / 模板市场 |
| 定价 | 免费版 ¥0（30 次 AI 创作/月）；Pro ¥29/月 或 ¥288/年（500 次/月） |
| 计费单位 | 每次经产品的 AI 对话生成计 1 次「AI 创作」，与 LLM 成本可对标 |
| 当前进展 | Web 创作台、三步 Agent、会员体系、公域「发现灵感」已上线可演示；**真实支付通道待接入** |

### 1.4 阶段性目标（12 个月）

1. **0–3 个月**：完成微信/支付宝支付与小红书单平台发布闭环；招募 50–100 名种子创作者，验证免费→Pro 转化与大纲定稿率。
2. **3–6 个月**：发现页内容运营与 SEO；扩展 2–3 个平台 OAuth；手机 App MVP（记灵感 + 与 Web 同步）。
3. **6–12 个月**：团队版/MCN 席位试点；行业大纲模板；创作者数据看板。

## 2. 问题与市场机会

### 2.1 创作者的核心痛点

| 痛点 | 表现 | 有感应对 |
|------|------|----------|
| **没选题** | 不知道写什么、发哪个平台、面向谁 | 灵感模式：结构化提问 + 已确认需求沉淀 |
| **方向不稳** | AI 一次生成全文，选题错了要整篇重写 | 大纲定稿门禁；创作模式严格按纲出稿 |
| **发布链路长** | 多工具写完再复制到各平台后台 | Pro 平台 OAuth 绑定；有感公域发布练手与曝光 |
| **状态分散** | 聊天记录难以回溯「当时定了什么」 | 作品 + 创作脉络侧栏（灵感 / 大纲 / 预览） |

### 2.2 市场机会

**宏观趋势**

- **创作者经济**：小红书、微信公众号、微博、抖音、快手、B 站等平台持续扶持 UGC，个人 IP 与垂类账号商业化成熟。
- **AI 渗透**：创作者普遍尝试 ChatGPT 等工具，但缺乏贴合「选题—结构—成稿」的垂直工作流产品。
- **多平台经营**：同一创作者常运营 2–3 个平台，对「平台适配 + 分发效率」需求上升。

**市场规模（需用第三方报告校准，以下为逻辑框架）**

| 层级 | 定义 | 估算逻辑（待填入数据） |
|------|------|------------------------|
| **TAM** | 中国内容创作者及相关运营人员中使用数字化工具的人群 | 行业报告：创作者规模 × 工具付费渗透率 |
| **SAM** | 以图文/短图文周更为主、有 AI 辅助写作需求的个人与小团队 | SAM ≈ TAM × 图文占比 × AI 意愿 |
| **SOM**（3 年） | 有感可触达并转化的付费创作者 | 种子平台（小红书+公众号）聚焦，首年目标【例：1–3 万注册，3–8% Pro 转化】 |

> **说明**：本初稿不引用未经核验的具体亿元市场规模数字；融资材料应附 QuestMobile、新榜、平台官方创作者报告等出处。

### 2.3 为什么不是「再做一个 ChatGPT 套壳」

有感的产品边界在代码与交互中已明确：

- 灵感模式**不写**大纲与正文；
- 大纲模式**未定稿不出**正文；
- 创作模式**不跳过**修改记录直接生成。

这是流程设计而非提示词堆砌，对应真实自媒体 SOP，构成品类差异的基础。

## 3. 解决方案与产品

### 3.1 产品定位

- **品类**：AI 自媒体创作助手  
- **Slogan**：帮你找灵感、写创作大纲，再按大纲生成能发的图文内容  
- **应用内主张**：灵感 · 大纲 · 创作  

### 3.2 三步创作方法论

```text
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌──────────┐
│  灵感模式   │ ──► │  大纲模式   │ ──► │  创作模式   │ ──► │   发布   │
│ 定选题/受众 │     │ 写创作大纲  │     │ 按纲出稿    │     │ 有感/平台│
└─────────────┘     └─────────────┘     └─────────────┘     └──────────┘
       │                    │                    │
       ▼                    ▼                    ▼
  已确认灵感           大纲定稿            标题/正文/标签
  （侧栏沉淀）      （pending/executed）   （内容预览）
```

**标准作业程序（SOP）**

| 步骤 | 用户动作 | 系统产出 |
|------|----------|----------|
| 01 | 新建作品（可含灵感推荐开场白） | 作品 + AI 会话线程 |
| 02 | 灵感模式对话，确认需求 | 侧栏「灵感」列表 |
| 03 | 大纲模式补充并定稿 | `outline_ready`，大纲摘要 |
| 04 | 创作模式生成与迭代 | 标题、正文、话题标签 |
| 05 | 发布 | 有感公域 / 第三方平台（Pro） |

同一件作品内可随时切换模式；界面、快捷键或自然语言指令均可，状态与侧栏即时同步。

### 3.3 创作台（Studio）

**布局**：左侧作品列表与分组 → 中间对话区 → 右侧「创作脉络」。

| 面板 | 功能 |
|------|------|
| 对话区 | 三模式共用；流式回复；工具调用可视化 |
| 灵感 | 已确认需求汇总，可增删改 |
| 创作大纲 | 待实现 / 已实现条目；定稿状态 |
| 内容预览 | 成稿标题、正文、标签 |
| 参考素材 | 上传或解析的文本/图片/网页参考 |

**辅助能力**：创意度调节（0.1–1.0）、作品分组、云端同步、参考素材解析、新建作品时的 AI 灵感推荐（1–3 条开场白）。

### 3.4 内容与社区：发现灵感

- **发现灵感**（`/content`）：浏览创作者公开发布的图文，按平台、形态、话题等筛选，用于选题参考与获客。
- **个人主页**：展示已发布作品、阅读总量、近 6 月发布趋势，支撑作者品牌。
- **飞轮**：创作 → 发布到有感 → 公域曝光 → 访客注册 → 成为创作者。

### 3.5 平台集成发布（Pro）

支持绑定并发布至：**小红书、微博、微信公众号、抖音、快手、哔哩哔哩**（OAuth 框架已具备，按平台分批上线运营配置）。

价值主张：**少一道复制粘贴**，在 Studio 完成三步后直达粉丝所在平台。

### 3.6 多端规划

- **Web**：当前主阵地，完整三步 + 设置 + 会员。
- **手机 App**（规划中）：通勤/睡前快速记灵感，回 Web 同作品续写大纲与成稿；账号与作品同步。

### 3.7 技术架构摘要

```text
浏览器 (Web)
    ↓ REST / OpenAPI
API 中间层 — 鉴权、作品、发布、订阅、上传、LangGraph 代理
    ├─ Postgres（业务数据）
    ├─ Redis（可选缓存）
    └─ Agent（LangGraph）— 灵感 / 大纲 / 创作子图
            └─ Postgres（会话 checkpoint）
```

- **主对话模型**：MiniMax（Anthropic 兼容端点）  
- **结构化任务**：DeepSeek（灵感推荐、灵感→大纲同步等）  
- **架构特点**：业务库与 AI 状态分离；OpenAPI 驱动前后端契约；便于后续 API 开放与白标扩展。

### 3.8 产品成熟度（诚实披露）

| 模块 | 状态 |
|------|------|
| 三步创作 + Web 创作台 | 可演示、开发环境生产可用 |
| 账号、作品、公域发布 | 可演示 |
| 会员与订单逻辑 | Beta（**模拟支付**） |
| 平台 OAuth 一键发布 | Alpha（需分平台配置） |
| 真实支付、内容审核、App 上架 | 建设中 |

## 4. 目标用户与客户细分

### 4.1 -primary 客户画像

**画像 A：个人自媒体创作者**

- 一人运营 1–3 个平台，周更或日更图文。
- 平台：小红书、公众号为主。
- 付费动机：省时间、少返工、稳定周更。

**画像 B：垂类内容运营者**

- 职场、知识、生活方式等垂类。
- 需要固定受众语气与结构模板。
- 付费动机：大纲复用、Pro 额度。

**画像 C：高频更新意向 Pro 用户**

- 每月数十次以上 AI 辅助创作。
- 免费版 30 次/月不足，驱动升级。

### 4.2 次级用户

| 用户 | 行为 | 价值 |
|------|------|------|
| 内容学习者 | 浏览发现灵感 | 获客、SEO |
| 品牌型创作者 | 经营主页与公开作品集 | 留存、口碑 |

### 4.3 典型场景

**场景 A：从零写一篇小红书种草**

灵感模式确认平台与选题 → 大纲模式定结构 → 创作模式出稿 → 发布到有感或小红书。

**场景 B：有方向、结构不清**

进入大纲模式，系统根据已确认灵感自动/sync 大纲条目 → 用户定稿 → 创作。

**场景 C：成稿后改语气/标题**

创作模式将修改记入待执行项 → 统一执行 → 生成执行摘要便于版本对比。

**场景 D：多端协作（规划）**

手机记灵感 → Web 续写与发布。

### 4.4 非目标用户（当前版本）

- 专业长视频剪辑全流程替代。
- 企业多层级审批与多席位权限（留待团队版）。
- 仅需排版、不需要 AI 的纯工具用户。

## 5. 竞争分析

### 5.1 竞争格局

| 类型 | 代表 | 与有感关系 |
|------|------|------------|
| 通用大模型对话 | ChatGPT、豆包、Kimi 等 | 替代：灵活但无作品 SOP；有感降低 Prompt 成本 |
| 一键 AI 写作 | 各类营销文案/写作 SaaS | 部分替代：更快但易返工；有感强调前期确认 |
| 多平台分发工具 | 排版、定时、数据统计类 | 互补：分发强、创作弱；有感做创作前端 |
| 笔记/协作 | Notion、飞书 | 弱替代：通用文档 ≠ 自媒体流水线 |

### 5.2 定位矩阵

```text
                    流程结构化程度高
                            ▲
                            │
                     ★ 有感 Yougan
                            │
    分发能力强 ◄────────────┼────────────► 分发能力弱
                            │
              多平台分发工具    通用 ChatGPT
                            │
                            ▼
                    流程结构化程度低
```

### 5.3 核心差异化（一句话）

> 有感不是「更快的 AI 写字机」，而是「把选题、结构和成稿分开确认的生产线」；确认之后再出稿，并连接到发布。

### 5.4 SWOT

| | 内容 |
|---|------|
| **S** | 三步方法论清晰；创作脉络可回溯；技术栈现代；公域发现具备生态雏形 |
| **W** | 支付未闭环；平台发布依赖运营配置；品牌与流量早期；App 未上架 |
| **O** | 创作者基数大；平台教育市场成熟；MCN/模板/B 端扩展 |
| **T** | 平台自带 AI；大模型降价压缩付费意愿；API 政策；同质化价格战 |

### 5.5 护城河建设路径

1. **流程与数据**：积累高转化大纲结构、选题模式，驱动推荐与模板。  
2. **生态**：发现页 UGC + 作者主页网络效应。  
3. **分发绑定**：OAuth 账号提高迁移成本。  
4. **垂类深耕**：先打透 1–2 个平台（建议小红书）再扩展。

## 6. 商业模式与定价

### 6.1 收入结构

| 来源 | 时间 | 说明 |
|------|------|------|
| Pro 订阅 | 当前主路径 | 月付/年付 |
| 免费版 | 获客 | 不收费，限额促转化 |
| AI 加购包 | 6 个月后 | 额度用尽增量收入 |

| 发现页广告/精选 | 可选 | 谨慎，避免伤害体验 |

### 6.2 套餐设计（现行价格）

| 套餐 | 价格 | AI 创作额度/月 | 核心权益 |
|------|------|----------------|----------|
| **免费版** | ¥0 | 30 次 | 灵感·大纲·创作全流程；作品分组；云端同步 |
| **Pro 创作者** | ¥29/月 | 500 次 | 上述全部 + 增强出稿承诺 + 优先响应 + 更多参考容量 + **平台集成发布** |
| **Pro 年付** | ¥288/年 | 500 次/月 | 约合 ¥24/月，提升 LTV、降低流失 |

**计费规则**：用户每发起一次经系统的 AI 创作对话计 **1 次**；超额提示升级 Pro 或次月重置（免费版按自然月）。

### 6.3 定价逻辑

- **30 次免费**：约够 1–2 篇完整三步体验，形成使用习惯。  
- **¥29/月**：低于许多创作者工具心理账户（「一杯咖啡」），便于冲动付费。  
- **500 次 Pro**：约日均 16 次，覆盖日更或多作品并行。  
- **平台发布绑 Pro**：强化付费理由，区别于纯聊天 AI。

### 6.4 单位经济模型

**变量定义**

| 符号 | 含义 | 初稿假设 |
|------|------|----------|
| P | Pro 月 ARPU | ¥29 |
| C | 单次 AI 创作边际成本（LLM+基础设施分摊） | ¥0.15–0.40（需实测） |
| U | Pro 用户月均使用次数 | 120 次（假设用量 24%） |
| r | 免费注册用户 → Pro 转化率 | 5%（种子期） |
| L | Pro 平均留存月数 | 8 个月 |
| CAC | 单注册用户获客成本 | ¥30（目标，待验证） |

**单用户毛利（Pro，月）**

```text
毛利 ≈ P - U × C
例：29 - 120 × 0.25 = ¥29（若 C=¥0.25 则 ≈ ¥29；需留通道费与固定成本）
```

**LTV（示意）**

```text
LTV_Pro ≈ P × L = 29 × 8 = ¥232（未扣退款与支付手续费）
```

**免费用户贡献**

```text
贡献 ≈ r × LTV_Pro - CAC
例：5% × 232 - 30 ≈ -¥18.4/注册（早期 CAC 高时可能为负，靠留存与年付改善）
```

> 正式融资须用上线后 90 天真实 CAC、转化率、次均 token 成本替换上表。

### 6.5 支付与订阅运营

- **目标流程**：下单 → 微信/支付宝 → 回调开通 → 账单与发票。  
- **当前**：模拟支付，仅用于产品与转化漏斗测试。  
- **退款**：支持订单退款，Pro 权益立即收回（降低付费顾虑）。  
- **续费**：支持到期取消续费；周期结束未续费自动降回免费版。

## 7. 市场进入与增长策略

### 7.1 获客（Acquisition）

| 渠道 | 动作 | KPI |
|------|------|-----|
| 落地页 + 能力页 | SEO、「AI 自媒体」「小红书 选题」等长尾词 | 注册转化率 |
| 发现灵感公域 | 精选内容、分享 slug、作者主页外链 | 访客 → 注册 |
| 种子创作者计划 | 邀请码、案例访谈、社群答疑 | 有效注册数 |
| 内容营销 | 三步 SOP 教程、对比「一问一文」 | 品牌搜索量 |
| 合作 | 自媒体训练营、MCN 工具采购试点 | B 端线索 |

### 7.2 激活（Activation）

**北极星激活事件**：注册后 24 小时内「创建 1 件作品 + 完成 1 次灵感模式对话」。

| 手段 | 说明 |
|------|------|
| 灵感推荐 | 新建作品时 1–3 条 AI 开场白，降低冷启动 |
| 空状态引导 | 建议「灵感 → 大纲 → 创作」路径 |
| 邮件/站内（规划） | 未完成首作品的用户召回 |

### 7.3 转化（Revenue）

| 触发点 | 机制 |
|--------|------|
| 额度将尽/用尽 | 会员页提示「已用 x/30」 |
| 大纲定稿后 | 引导进入创作，体验完整价值再触达付费墙 |
| Pro 功能预览 | 平台集成页展示绑定价值 |
| 年付折扣 | ¥288/年降低价格敏感 |

**目标转化率（种子期假设）**：注册 → Pro **3–8%**。

### 7.4 留存（Retention）

| 机制 | 说明 |
|------|------|
| 作品云同步 | 换设备续写 |
| 创作脉络资产 | 切换作品即可回溯决策 |
| 分组管理 | 多栏目/系列并行 |
| 平台账号绑定 | 提高迁移成本 |
| 发现页曝光 | 发布获得阅读反馈 |

### 7.5 核心 KPI 体系

| 阶段 | 指标 |
|------|------|
| 获客 | UV、注册率、CAC |
| 激活 | 首作品率、首对话率、D1 留存 |
| 参与 | 大纲定稿率（M2）、月活跃作品数 |
| 变现 | 免费→Pro 转化率、MRR、ARPU、退款率 |
| 留存 | M3 留存、Pro churn、年付占比 |
| 生态 | 公开发布率、发现页 PV/UV、篇均阅读 |

## 9. 财务预测

> **重要**：以下为**说明性三年模型**，用于演示逻辑一致性；所有数字须在真实支付上线后用运营数据校准。

### 9.1 关键假设

| 假设项 | Y1 | Y2 | Y3 |
|--------|----|----|-----|
| 累计注册用户 | 20,000 | 80,000 | 200,000 |
| 期末月活（MAU） | 4,000 | 18,000 | 45,000 |
| Pro 付费用户（期末） | 600 | 3,200 | 10,000 |
| Pro 月 ARPU（混合年付） | ¥27 | ¥27 | ¥28 |
| 免费→Pro 转化率（年均） | 4% | 5% | 5.5% |
| Pro 月流失率 | 8% | 6% | 5% |
| 单次 AI 成本 C | ¥0.28 | ¥0.22 | ¥0.18 |
| Pro 月均 AI 次数 U | 100 | 110 | 120 |
| 获客 CAC（混合） | ¥35 | ¥28 | ¥22 |

### 9.2 收入预测（人民币，万元）

| 项目 | Y1 | Y2 | Y3 |
|------|----|----|-----|
| 订阅收入（Pro） | 58 | 310 | 980 |
| 加购包/其他 | 0 | 15 | 80 |
| **营业收入合计** | **58** | **325** | **1,060** |

*Y1 订阅测算逻辑示例：平均付费用户约 350人 × ¥27 × 12月 ≈ 113万理论上限；初稿取 58万反映前半年支付未上线、转化爬坡。*

### 9.3 成本与费用（万元）

| 项目 | Y1 | Y2 | Y3 |
|------|----|----|-----|
| LLM 与云服务（变动） | 18 | 72 | 180 |
| 研发人力 | 120 | 180 | 240 |
| 运营与市场 | 60 | 120 | 200 |
| 管理与杂项 | 20 | 35 | 50 |
| **费用合计** | **218** | **407** | **670** |

### 9.4 损益概要（万元）

| 项目 | Y1 | Y2 | Y3 |
|------|----|----|-----|
| 营业收入 | 58 | 325 | 1,060 |
| 费用合计 | 218 | 407 | 670 |
| **营业利润** | **-160** | **-82** | **+390** |

*Y1–Y2 亏损符合早期产品研发与获客投入；Y3 盈亏平衡依赖 MAU 与 Pro 规模达到上表假设。*

### 9.5 现金流与融资

- Y1 净现金消耗主要来自人力与获客，需融资或自有资金支持至 **MRR 覆盖变动成本** 的拐点（模型约在 Y2 中后期，以实测为准）。  
- 建议设立 **12–18 个月跑道** 再规划下一轮融资或盈利路径。

### 9.6 敏感性分析（简述）

| 变量恶化 | 影响 |
|----------|------|
| 转化率 -2pp | MRR 下降约 30–40%，跑道缩短 |
| C 上升 30% | 毛利率压缩，需调价或降次 |
| CAC 上升 50% | LTV/CAC < 1，需收缩投放 |
| Pro 流失 +3pp | LTV 显著下降 |
