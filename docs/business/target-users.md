# 目标用户与使用场景

## 主目标用户（Primary）

### 1. 个人自媒体创作者

- **特征**：一人运营 1–3 个平台账号，周更或日更图文/短内容。
- **典型平台**：小红书、微信公众号、微博（产品文案与 `SUPPORTED_PLATFORMS` 一致）。
- **痛点**：选题枯竭、写法摇摆、AI 出稿不可控。
- **有感契合点**：定方案对话结构化整理；制作环节定计划后再出稿；作品按系列/栏目分组。

### 2. 职场/垂类内容运营者

- **特征**：为品牌或个人 IP 产出职场、知识、生活方式等垂类内容。
- **场景示例**（来自开屏 `nextStepSuggestions` 与产品文案）：
  - 「想更新自媒体但还没想好写什么」
  - 「想做职场类内容，不确定选题方向」
  - 「还没定发小红书还是公众号」
- **有感契合点**：在定方案阶段锁定受众、语气、平台；制作环节可指定「受众 25–35 岁职场女性、语气专业好读」等。

### 3. 高频更新的 Pro 意向用户

- **特征**：每月需要数十次以上 AI 辅助创作。
- **有感契合点**：免费版每月 30 次额度；Pro 500 次 + 平台集成发布（套餐文案）。

## 次级用户（Secondary）

### 4. 内容学习者 / 选题研究者

- **行为**：浏览「发现灵感」（`/content`）看他人公开作品，找选题参考。
- **价值**：公域内容 + 筛选维度（平台、形态、话题等，`discover-taxonomy`）构成轻量内容社区。

### 5. 希望建立个人品牌的创作者

- **行为**：维护个人主页（头像、封面、简介）、发布公开内容、展示阅读数据。
- **指标**：已发布篇数、总阅读量、近 6 月发布趋势（`ACCOUNT_PAGE` 文案）。

## 用户分层（按产品使用深度）

| 层级 | 行为特征 | 对应产品模块 |
|------|----------|--------------|
| 访客 | 浏览首页、能力页、公开内容 | 营销页、`/content` |
| 注册用户 | 登录、创建作品、使用免费额度 | Auth、Studio、Subscription |
| 活跃创作者 | 完成灵感→创作全流程，发布到有感 | Works、Publications、Discover |
| 付费用户 | Pro 订阅、平台 OAuth、高额度 | Billing、Integrations |
| 多设备用户 | 手机记灵感、Web 写全文（规划） | Mobile 文案、账号同步 |

## 典型使用场景

### 场景 A：从零写一篇小红书种草

1. 新建作品，可选开屏 7 条下一步建议。
2. 定方案对话确认：平台=小红书、选题=护肤种草、受众与语气。
3. 说「开始制作」，production 子图生成并编排制作计划（含封面风格、正文结构）。
4. 制作按计划生成标题、正文、标签，在「作品」面板修改。
5. 发布到有感「发现灵感」或（Pro + OAuth 配置后）同步小红书。

### 场景 B：有方向但结构不清

1. 跳过部分方案讨论，直接说「开始制作」。
2. production 子图根据现有方案自动编排制作任务（`schedulePlan`）。
3. 用户补充后再次触发制作，产出 preview。

### 场景 C：成稿后反复改语气

1. 已有 preview 且 productionPlan 已编排。
2. 用户提出「语气更口语」「标题更吸引人」→ 记入待执行项后统一执行。
3. `complete_execution` 后更新「已实现」条目与执行摘要。

### 场景 D：通勤记灵感，回家写全文

1. 手机 App 快速记录（产品规划，依赖下载链接配置）。
2. Web 创作台同一账号、同一件作品继续创作与出稿。

## 非目标用户（当前版本）

- 长视频剪辑与重度非图文工作流（抖音/ B 站以「内容形态支持」为主，非专业剪辑替代）。
- 企业多账号权限与审批流（当前为单用户账号模型）。
- 无需 AI、仅需排版发布的纯工具用户。

## 商业计划书：市场规模叙述建议

可结合 TAM/SAM/SOM 框架自行填入数字；本产品代码层面可支撑的市场叙述为：

- **SAM**：中国图文自媒体创作者 + 小型 MCN 运营者中使用 AI 辅助写作的人群。
- **SOM**：先聚焦小红书/公众号周更个人创作者，以 Web Studio + 订阅变现验证 PMF。

（具体市场规模需外部行业数据，本文档仅定义用户边界。）
