# 竞争与替代方案

本文档基于产品自述与功能设计归纳竞争格局，**非第三方市场调研报告**。商业计划书中的市场分析章节应补充行业数据与竞品财务信息。

## 竞争维度

有感在三个维度建立差异：

1. **流程**：三步确认 vs 一次性全文。
2. **资产**：创作脉络侧栏 vs 仅聊天历史。
3. **分发**：多平台 OAuth + 公域发现 vs 纯写作工具。

---

## 替代方案分类

### A. 通用大模型对话（ChatGPT、豆包、Kimi 等）

| 对比项 | 通用对话 | 有感 |
|--------|----------|------|
| 使用方式 | 自由 Prompt | 结构化三模式 SOP |
| 选题阶段 | 用户自拟 Prompt | 灵感模式专用工具与确认流 |
| 制作计划 | 靠 Prompt 一次生成 | 创作模式内创意总监 + 定稿门禁 |
| 作品管理 | 多会话无「作品」概念 | Work + 分组 + thread 绑定 |
| 发布 | 无 | 有感公域 + 平台 OAuth |
| 计费 | 各家订阅/按 token | 按「次」简化理解 |

**有感优势叙述**：降低 Prompt 工程门槛，用产品流程约束 AI 行为，适合**反复周更**而非偶发提问。

**有感劣势**：灵活性低于开放对话；重度 Power User 可能偏好自建 GPTs。

---

### B. 一键 AI 写作 / 营销文案工具

| 对比项 | 典型 AI 写作工具 | 有感 |
|--------|------------------|------|
| 产出速度 | 快，一问一文 | 慢，多轮确认 |
| 返工成本 | 选题错则全文重写 | 早期纠偏 |
| 平台适配 | 常提供模板 | 从灵感阶段嵌入平台/受众 |
| 自媒体场景 | 泛营销 | 文案强调小红书、公众号 |

**有感优势**：适合**选题不确定**阶段的创作者。

**有感劣势**：追求「5 分钟出一篇」的用户可能流失。

---

### C. 自媒体多平台分发工具（壹伴、新榜、各平台官方助手等）

| 对比项 | 分发工具 | 有感 |
|--------|----------|------|
| 核心能力 | 排版、定时、数据统计 | AI 创作流程 |
| 创作 | 弱或外挂 | 核心 |
| 分发 | 强 | OAuth 发布（建设中） |

**关系**：互补大于替代。有感可定位为「创作前端」，与分发工具集成或逐步补齐发布。

---

### D. 笔记与协作（Notion、飞书文档）

| 对比项 | 笔记工具 | 有感 |
|--------|----------|------|
| AI | 近年增强 | 原生三模式 |
| 结构 | 自由文档 | 灵感/计划/成稿字段化 |
| 社区 | 弱 | 发现灵感 |

**有感优势**：专为自媒体流水线设计。

**有感劣势**：通用知识管理、团队协作不如 Notion/飞书。

## SWOT（供商业计划引用）

### Strengths 优势

- 清晰的三步方法论与产品内嵌 SOP。
- 作品 + 创作脉络，状态可回溯。
- 技术栈现代（LangGraph、双库、OpenAPI），利于快速迭代。
- 公域发现 + 主页，具备生态扩展基础。

### Weaknesses 劣势

- 支付仍为模拟，商业化未闭环。
- 六平台 OAuth 依赖运营配置，一键发布体验未全面验证。
- 手机 App 依赖外部配置，多端故事未完全落地。
- 品牌与流量处于早期。

### Opportunities 机会

- 自媒体创作者基数大，AI 渗透率上升。
- 小红书等平台创作者教育市场成熟。
- MCN、知识付费可采购「标准化创作流程」工具。
- 行业模板、垂类计划模板库可增值。

### Threats 威胁

- 平台自带 AI 写作（公众号助手等）挤压独立工具空间。
- 大模型降价导致「按次订阅」价值感知下降。
- 平台 API 收紧影响 OAuth 发布。
- 同质化 AI 写作产品价格战。

---

## 护城河建设建议（写入商业计划「战略」）

1. **流程数据**：积累已定稿计划、高转化选题结构，训练推荐与模板。
2. **生态**：发现页 UGC 与作者主页形成网络效应。
3. **分发绑定**：平台账号连接提高迁移成本。
4. **垂类品牌**：先打透 1–2 个平台（如小红书）再扩展。

---

## 一句话竞争陈述

> 有感不是「更快的 AI 写字机」，而是「帮自媒体人把选题、结构和成稿分开确认的生产线」；在确认之后再出稿，减少返工，并一路连接到发布。
